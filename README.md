# CSES Problem Set: Rust Edition

Solutions to the [CSES Problem Set](https://cses.fi/problemset/) written in Rust.

## Project Structure

- **`src/bin/`** – One binary per CSES problem (e.g., `weird_algorithm.rs`, `missing_number.rs`)
- **`src/lib.rs`** – `FastIO`: a zero-allocation I/O struct that reads all input upfront and writes to an in-memory buffer, flushed once at the end
- **`notes/`** – Per-problem notes covering pattern, technique, and key insight
- **`bundles/`** – Monolithic single-file submissions ready for CSES (generated by `bundle.sh`)

## Common Commands

### Run a solution

```bash
cargo run --bin missing_number
```

### Test all solutions

```bash
cargo test
cargo test --bin weird_algorithm   # single binary
cargo test --quiet                 # less verbose
```

### Format and lint

```bash
cargo fmt
cargo clippy --all-targets
```

### Bundle for CSES submission

```bash
bash bundle.py missing_number
# Output: bundles/missing_number.rs
```

## FastIO

All solutions use `FastIO` from `src/lib.rs` for I/O. It reads stdin entirely upfront, parses integers directly from bytes, and accumulates output in a `Vec<u8>` that is flushed once via `stdout`. This avoids per-token allocation and syscall overhead.

```rust
fn solve(io: &mut FastIO) {
    let n = io.next_u64();
    io.write_u64(n * 2);
    io.newline();
}

fn main() {
    let mut io = FastIO::new();
    solve(&mut io);
    io.flush();
}
```

Tests use `FastIO::from_bytes` and `into_output` to exercise the full I/O path:

```rust
fn run(input: &[u8]) -> Vec<u8> {
    let mut io = FastIO::from_bytes(input);
    solve(&mut io);
    io.into_output()
}

#[test]
fn test_example() {
    assert_eq!(run(b"3\n"), b"6\n");
}
```
